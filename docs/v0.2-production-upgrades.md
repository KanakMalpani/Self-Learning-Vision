# v0.2 Production Upgrades

Self-Learning Vision keeps the default path local and lightweight. The v0.2 production layer adds optional adapters for stronger face embeddings, vector search, and background jobs.

## Optional Face Models

Default behavior remains:

```env
EMBEDDING_PROVIDER=auto
MODEL_CACHE_DIR=/app/data/models
```

Use InsightFace only when the production extras and local model cache are prepared:

```bash
cd apps/api
pip install -r requirements-production.txt -c constraints.txt
python scripts/download_models.py --provider insightface --download
```

Then configure:

```env
EMBEDDING_PROVIDER=insightface
```

If InsightFace or its models are unavailable, the API reports provider health and falls back to the local provider where possible.

## Optional Vector Store

Default:

```env
VECTOR_STORE=local_json
```

Production:

```env
VECTOR_STORE=pgvector
```

The new `VectorStore` interface supports `upsert`, `search`, `delete`, and `list_for_entity`. The local JSON implementation is useful for tests and demos. The pgvector implementation is for Postgres-backed deployments.

## Optional Background Jobs

Default:

```env
JOB_BACKEND=inline
```

Production:

```env
JOB_BACKEND=celery
REDIS_URL=redis://redis:6379/0
```

The new `JobBackend` interface supports inline execution for local development and Celery for heavier clustering, replay, and evaluation jobs.

## Production Docker Profile

Run the optional production stack with:

```bash
docker compose -f docker-compose.yml -f docker-compose.production.yml --profile production up --build
```

This switches Postgres to pgvector and adds Redis plus a Celery worker. The default `docker compose up --build` path remains unchanged.

## Health And Readiness

Use:

- `GET /ready`
- `GET /api/v1/providers/health`

Readiness shows configured optional features. Provider health reports whether InsightFace dependencies and model cache are available.

## E2E And Pre-Commit

Frontend E2E smoke tests use Playwright:

```bash
cd apps/web
npm run test:e2e
```

Install pre-commit locally when you want commit-time checks:

```bash
pip install pre-commit
pre-commit install
```

The hooks run API formatting/lint checks and the frontend typecheck. Playwright remains a CI/manual check because browser tests are heavier.
